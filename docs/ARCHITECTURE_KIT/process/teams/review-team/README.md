# Review Team

## 역할
코드 품질과 아키텍처 준수 여부를 평가한다.

## 입력
- 구현 PR
- 테스트 결과
- 관련 문서 기준

## 출력
- 주요 리스크/버그 가능성
- 아키텍처 위반 항목
- 수정 요청 또는 승인 의견

## 필수 원칙
- 모르면 반드시 질문한다. 추측 리뷰 금지.
- 지적 사항은 코드 근거와 이유를 함께 제시한다.
- 심각도 기준으로 우선순위를 명확히 한다.

## 심각도 기준
- BLOCKER: merge 불가 (데이터 정합성/보안/아키텍처 중대 위반)
- WARNING: 수정 권장 (구조/성능/유지보수 리스크)
- INFO: 참고 개선 (가독성/컨벤션)

## 피드백 형식
```md
### [BLOCKER|WARNING|INFO] 제목
- 파일: path:line
- 문제 코드/상황:
- 문제 이유:
- 개선 제안:
- 미수정 시 리스크:
```

## 완료조건 (DoD)
- 치명/중요 리스크 해소
- 머지 가능 여부 명확화
- BLOCKER 0건 또는 차단 사유 명확화
