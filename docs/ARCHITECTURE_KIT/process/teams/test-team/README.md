# Test Team

## 역할
머지 직전 최종 검증을 수행한다.

## 입력
- 리뷰 통과 후보 브랜치
- 테스트 시나리오

## 출력
- 테스트 리포트(pass/fail)
- 실패 시 재현 절차

## 검증 순서
1. 변경사항 분석 (영향 모듈/영향 테스트 식별)
2. 단위 테스트 검증
3. 빌드/컴파일 검증
4. E2E/API 흐름 검증
5. 결과 종합 보고

## 실패 보고 형식
```md
### 실패 케이스
- 테스트명:
- 기대값:
- 실제값:
- 원인:
- 변경 기인 여부:
- 재현 절차:
```

## 완료조건 (DoD)
- 필수 테스트 전부 통과
- 배포/머지 차단 이슈 없음
- 실패 항목은 기대값/실제값/원인/재현 절차가 모두 기록되어 있다
