# Design Team Index

## 1) Who
설계 책임자와 도메인 대표.

## 2) When
- 작업 시작 직전(요구사항 수신 직후)
- 기존 설계 변경이 필요한 시점
- 의존성/에러 모델/레이어 경계 재정의가 필요한 시점

## 3) What
### 입력
- 요구사항
- 제약사항(일정, 호환성, 플랫폼 제약)
- 기존 아키텍처 문서

### 출력
- 설계 결정서(무엇/왜/범위)
- 변경 영향도(모듈/레이어)
- 리스크 및 트레이드오프
- 구현 체크리스트

### 전달 기준
- implement-with-tdd-team로 넘길 수 있는 형식인지
- 미결사항이 명시되어 있는지
- 공통 DoD 충족 여부
