# Implement With TDD Team

## 역할
설계 결정을 코드와 테스트로 구현한다.

## 입력
- design-team 산출물
- 관련 아키텍처 문서

## 출력
- 코드 변경
- 테스트(Unit/Reducer/ViewModel 등)
- 문서 동기화(필요 시)

## 필수 원칙
- 모르면 반드시 질문한다. 추측으로 구현하지 않는다.
- Red -> Green -> Refactor 순서를 지킨다.
- 구현과 테스트를 함께 제출한다.

## TDD 실행 규칙
1. Red: 실패 테스트를 먼저 작성하고 실제로 실패를 확인한다.
2. Green: 테스트를 통과시키는 최소 구현만 작성한다.
3. Refactor: 동작 유지 상태에서 구조/가독성을 개선한다.

## 산출물 템플릿
```md
## 구현 결과
- 변경 파일:
- 테스트 파일:

## 테스트 결과
| 구분 | 실행 | 성공 | 실패 |
|------|------|------|------|

## 실패 상세 (있는 경우)
- 케이스:
- 기대값:
- 실제값:
- 원인:
- 조치:
```

## 완료조건 (DoD)
- 신규 로직 테스트 통과
- 기존 동작 회귀 없음
- 문서 규칙 위반 없음
- 실패 케이스의 기대값/실제값/원인이 기록되어 있다
