# Implement Team Index

## 1) Who
구현/테스트를 책임지는 팀.

## 2) When
- 설계 산출물을 받을 때
- 기존 결정이 바뀌어 코드 수정이 필요한 경우

## 3) What
### 입력
- design-team 산출물
- 관련 아키텍처 문서

### 출력
- 코드 변경
- 테스트(정상/실패 경로)
- 문서 동기화 내역

### 전달 기준
- Red -> Green -> Refactor 순서를 따랐는지
- 최소 실행 결과와 근거가 PR 산출물에 남아 있는지
- 다음 리뷰 게이트를 통과할 수 있는 포맷인지
